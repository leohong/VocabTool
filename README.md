# 🧠 極限單字特訓系統

> 一款基於「主動回想」、「動態錯題懲罰」與「間隔重複」的純前端單字記憶 Web 應用程式。

🔗 **線上即用網址**：[https://leohong.github.io/VocabTool/](https://leohong.github.io/VocabTool/)
📦 **內建字庫下載**：[國中 2000 單字庫](https://github.com/leohong/VocabTool/raw/main/2000_單字庫.txt) | [高中 7000 單字庫](https://github.com/leohong/VocabTool/raw/main/7000_單字庫.txt)

專為高強度的長期單字記憶計畫（如征服國中 2,000 單字、高中 7,000 單字）而設計。透過極簡的單一 HTML 檔案架構，結合雙階段流水線訓練、嚴格的拼寫容錯機制、動態間隔重複演算法，以及直覺的視覺化狀態燈號，幫助學習者打破遺忘曲線，將單字真正刻進長期記憶與肌肉記憶中。

---

## 🚀 如何開始 (Getting Started)

本系統為單一 HTML 檔案架構 (Single File Application)，支援線上免安裝即用，亦可下載至本地執行。

### 💻 步驟 1：開啟系統
*   **線上即用 (推薦)**：直接造訪 **[線上特訓系統網址](https://leohong.github.io/VocabTool/)**。
*   **本地伺服器執行 (強烈推薦)**：雙擊執行專案根目錄的 `start_dev_server.bat`，它會自動檢查並關閉之前卡死的伺服器行程，並在新視窗中啟動本機伺服器。啟動後直接在瀏覽器造訪 **[http://localhost:8001](http://localhost:8001)**。
*   **雙擊直接開啟**：直接雙擊 `index.html` 以瀏覽器開啟（注意：部分功能如 API 字典查詢，在部分瀏覽器的安全限制下可能會被阻擋，故強烈推薦使用上面的伺服器啟動器）。

### 📥 步驟 2：下載官方單字庫
系統首頁未載入大量單字，建議先下載以下官方預載字庫檔：
*   👉 **[國中 2000 單字庫 TXT](https://github.com/leohong/VocabTool/raw/main/2000_單字庫.txt)**
*   👉 **[高中 7000 單字庫 TXT](https://github.com/leohong/VocabTool/raw/main/7000_單字庫.txt)**

### 🔌 步驟 3：匯入字庫開始特訓
1. 在系統首頁最下方的「檔案管理與備份」區塊，找到 **`⬆️ 字`** 按鈕（匯入字典）。
2. 點擊 **`⬆️ 字`**，並選取剛才下載的 `.txt` 單字庫檔案。
3. 看到「字典匯入成功！」的彈窗提示後，即可在上方點擊 **「⚡ 發動今日特訓」** 開始進行特訓！

## 🎮 主介面與操作指南

### 1. 指揮中心按鈕
*   **⚡ 發動今日特訓**：開始當天設定的每日新單字特訓。特訓中會自動安插今日到期的歷史「幽靈單字 👻」進行突襲。
*   **🔥 降溫：錯題大會考**：從當前錯題中隨機抽取最多 50 個字進行純拼寫盲測，幫助快速消除錯題債務。
*   **🏛️ 深度：歷史隨機抽查**：隨機抽取最多 50 個已封存於歷史殿堂的單字進行純拼寫抽驗，主動檢測長期記憶。
*   **🎧 聽讀：聽音背單字**：進入聽音背單字特訓播放器。支援播放自訂範圍的單字、拼讀字母、朗讀中文釋義與例句，並配備眼睛隱藏遮蔽（盲聽）功能。
*   **🔍 預覽今日特訓新單字**：在發動特訓前，可以先預覽今日的單字清單與發音。
*   **🗂️ 預覽全字庫單字**：開啟全字庫預覽面板，提供關鍵字篩選、分頁瀏覽，以及即時編輯與徹底刪除功能。
*   **📖 查字典與新增單字**：整合線上英語字典與翻譯 API，查詢後可直接調整內容並手動新增至字庫中。

### 2. 進度與封存統計
*   **總進度覆蓋**：顯示已學習的單字與字庫總數比例。
*   **🏛️ 歷史殿堂**：點擊開啟視窗，查看所有已封存單字的下次突襲日期、總失誤次數，以及是否已達到「永久免疫」。

### 3. 系統設定
*   **🗄️ 字庫管理**：切換與新增自訂字庫。
*   **🎯 每日新字數**：每天特訓的基礎新字量（預設為 50 字）。
*   **👻 每日幽靈數**：每天最多允許被歷史幽靈突襲的單字上限（預設為 10 字），避免負擔過重。

### 4. 檔案管理與備份
*   **`⬇️ 字` / `⬆️ 字`**：匯出與匯入純單字庫 TXT 檔。
*   **`⬇️ 殿` / `⬆️ 殿`**：匯出與匯入歷史殿堂 TXT 檔（具備轉為全新字庫重新特訓與測試之功能）。
*   **`⬇️ J` / `⬆️ J`**：全系統 JSON 完整備份與還原。匯出時會下載包含所有字庫單字與進度、每日目標等設定的完整備份檔（檔名如：`極限完整備份_20260712_025340.json`），且完全向下相容舊版單一字庫備份格式。

## 📚 資料匯入格式 (Data Format)

你可以透過介面中的「匯入字典 (TXT)」功能，快速載入自己的單字庫。TXT 檔案的內容必須符合以下正規表達式格式：

### 📝 字典 TXT 格式範例：
必須包含 `=== 特訓完整字庫 ===` 開頭，單字行格式為 `編號. [詞性] 英文 --> 中文`。
您可以選擇使用 `||` 符號在中文釋義後方追加例句（可選，無例句時直接省略 `||` 及其後內容即可）。

> [!TIP]
> **聽讀靜音優化**：例句格式建議採用 `英文例句 (中文翻譯)` 或 `英文例句 （中文翻譯）`，系統在聽讀朗讀例句時，會自動偵測並將中文括號部分過濾靜音，僅朗讀英文例句，以維持盲聽專注度。

```text
=== 特訓完整字庫 ===

1. [n.] system --> 系統 || A computer system is complex. (電腦系統很複雜。)
2. [v.] acknowledge --> 承認/確認 || I acknowledge my mistake. (我承認我的錯誤。)
3. [adj.] extreme --> 極端的
```

### 🏛️ 歷史殿堂 TXT 匯入格式範例：
必須包含 `=== 歷史殿堂單字個人紀錄 ===` 開頭，單字行格式為 `編號. 錯誤次數: X次 | [詞性] 英文 --> 中文`。與字典檔相同，您可以使用 `||` 符號在中文釋義後方追加例句（可選，無例句時直接省略即可）。

```text
=== 歷史殿堂單字個人紀錄 ===

1. 錯誤次數: 3次 | [n.] system --> 系統 || A computer system is complex. (電腦系統很複雜。)
2. 錯誤次數: 5次 | [adj.] extreme --> 極端的
```

當您匯入這些檔案時，系統具有安全的防護機制：
*   **點擊 `⬆️ 殿` (匯入歷史殿堂)**：系統會直接將此個人紀錄匯入為目前的 **「歷史殿堂」**（供幽靈突襲用），不影響目前正在特訓的字庫與進度。
*   **點擊 `⬆️ 字` (匯入字典檔)**：系統支援自動辨識「特訓完整字庫」及「歷史殿堂」兩種格式。
    *   若是「特訓完整字庫」格式：將直接**覆寫**當前的字庫。
    *   若是「歷史殿堂」格式：系統會將這批歷史單字**去重合併追加**至目前字庫中，保留原本字庫中已有的字。
    *   **請注意**：不論是哪種格式，此匯入動作都**絕對不會清洗或重置您當前的學習天數與進度**。如果您想將這批殿堂單字作為全新的字庫重新開始第 1 天的特訓，請先點擊 `➕庫` 建立一個新資料庫後，再點擊 `⬆️ 字` 進行匯入，這可完美確保您主進度字庫的安全。

## 💡 使用建議 (Usage Tips)

1.  **每日單字量**：建議依個人吸收能力，將「每日新字」設定在 `30 ~ 50` 之間。隨時間推移，歷史殿堂到期的「幽靈單字 👻」會逐漸增加，細水長流才能維持打卡不中斷。
2.  **關注燈號**：當介面亮起 🟡 黃燈或 🔴 紅燈時，請暫停推進新進度，優先使用「降溫：錯題大會考」清除錯題本中的單字，避免記憶過載。
3.  **定期備份**：由於資料完全儲存於本地瀏覽器快取中，清除瀏覽器快取會導致資料遺失。請務必養成定期點擊「JSON全備份匯出 (`⬇️ J`)」的習慣。

## ✨ 核心特色 (Features)

*   ⚡ **雙階段流水線訓練 (Two-Stage Pipeline)**
    *   **第一關 [快速篩選]**：以閃卡（Flashcard）模式呈現英文與中文。認識按 `→` 鍵（或右滑）；不認識或猶豫按 `←` 鍵（或左滑）。不熟單字會立即被送入**當前錯題集中營**。
    *   **第二關 [強制盲測]**：第一關篩選結束後，系統會將所有單字打亂，強制隱藏英文，僅顯示中文解釋與字母長度，要求你**全手動拼寫**出完整單字。
*   🔥 **強迫記憶與嚴格糾錯 (Strict Punishment System)**
    *   **手滑警告**：第一次拼寫錯誤視為手滑，給予警告不扣分。
    *   **失憶判定**：第二次拼寫錯誤判定為記憶斷裂。系統會強制鎖定並以亮綠色顯示答案，**強迫你對著正確答案完整輸入一次（強迫重抄）**以重新建立神經鏈結。同時該字連續答對次數歸零，錯題累加。
*   📉 **雙倍消除法 (Double Elimination)**
    *   移出當前錯題的標準為：必須在盲測中**連續拼對指定的目標次數**。
    *   消除目標次數計算公式為：`Min(該字錯誤次數 × 2, 6)` 次。拼錯越多次的單字，需要連續拼對越多次才能消除。
*   🏛️ **歷史殿堂與間隔重複 (Spaced Repetition)**
    *   錯題完成雙倍消除後，晉升至**歷史殿堂**，系統會依間隔天數（`7` ➔ `21` ➔ `60` ➔ `180` 天）自動將其作為**「幽靈單字 👻」**安插回每日特訓進行突襲抽查。
    *   幽靈單字若答對則拉長間隔，若答錯兩次會被**踢出殿堂**送回錯題集重修。通過 180 天抽查後獲得**「永久免疫 🛡️」**。
*   📊 **動態溫度計監控 (Thermometer Indicator)**
    *   隨時監控「錯題負債」數量，依據大腦記憶塞車程度給予🟢綠（良好）、🟡黃（記憶塞車）、🔴紅（過載警告）三種燈號提示。
    *   過載時可一鍵發動「降溫特訓（隨機 50 字大會考）」，優先清空錯題負債。
*   💾 **無後端與中斷續傳 (Zero-Backend & Session Auto-Save)**
    *   所有進度、字典與學習狀態皆儲存於瀏覽器的 `localStorage` 中。
    *   每次操作皆會自動存檔，若不小心關閉瀏覽器，下次開啟時可點擊「繼續未完成進度」進行**中斷續傳**。
*   📂 **多字庫管理與智慧分流 (Multi-DB & Smart Routing)**
    *   支援建立多組獨立學習進度的字庫（如 `vocab_2000`、`vocab_7000`）。
    *   提供 TXT、JSON 格式的字典與紀錄匯出/匯入。
    *   **智慧分流**：匯入舊錯題時，可選擇一鍵轉移至【歷史殿堂】進行間隔突襲，大幅減輕初期雙倍消除的拼寫負擔。
*   📖 **智慧字典查詢與手動加字 (Dict Lookup & Custom Word)**
    *   **免代理高穩定查詢**：整合免金鑰且支援 CORS 的 Free Dictionary API 與 MyMemory 翻譯 API，直接從瀏覽器查詢英文釋義、詞性與例句，並自動翻譯為繁體中文，避免 Cloudflare 代理阻擋。
    *   **智慧例句退路機制**：若查詢的釋義無預設例句，會自動跨釋義借用同單字的例句；若 API 中完全無例句（如學術字 `algorithm`），則會以英文定義構造出 `A(n) [word] is defined as: [definition]` 的學術例句並同步翻譯，確保盲測時必定有高品質遮蔽提示。
    *   **手動加字定位**：支援自訂編輯單字屬性，並可選擇追加至「字庫最末端」或插至「當前特訓進度」尾端，以便今日特訓立即生效。
*   🗂️ **全字庫單字預覽與分頁過濾 (Full Database Preview)**
    *   提供「預覽全字庫單字」面板，支援對目前字庫中的所有單字進行關鍵字即時模糊篩選（可依單字、釋義或詞性進行過濾）。
    *   配備每頁 50 字的分頁瀏覽機制，在達數千字的字庫下依舊保持流暢不卡頓。
*   ✏️ **即時編輯與徹底刪除 (Card Editing & Deletion)**
    *   在 **[快速篩選]**、**[強制盲測]**、**[今日新單字預覽]** 以及 **[全字庫單字預覽]** 等介面中，單字卡與清單均整合了 `✏️ 編輯` 與 `🗑️ 刪除` 按鈕。
    *   支援在特訓中即時修改拼寫錯誤或中文解釋，並在背景自動同步更新目前正在特訓的單字緩衝佇列（Queue）。
    *   刪除單字時會即時自動跳過該字，特訓流程安全流暢。
*   🎧 **聽音特訓播放器與盲聽模式 (Audio Dictation Player)**
    *   **多重單字來源**：支援選取今日進度、錯題集中營、歷史殿堂以及全字庫單字進行聽寫背單字。
    *   **客製化播放設定**：提供單字重複播放次數、單字間停頓時間、例句/翻譯朗讀開關，以及「拼讀字母間的停頓時間」選項（支援 `0 秒` 連讀至 `0.5 秒` 停頓，預設 `0 秒` 連讀）。
    *   **發音與語速最佳化**：中文發音預設自動選取 Traditional Chinese 台灣國語/香港發音人；字母發音速率保持與單字讀速同步。
    *   **眼睛關閉盲聽遮蔽**：一鍵開啟盲聽模式，遮蓋單字拼寫與中文釋義，拼讀字母時網頁上會有同步高亮逐字放大的 Premium 視覺微動畫，大幅提升聽寫專注度。
    *   **防鎖屏休眠**：整合 Wake Lock API，防止行動裝置背景播放時螢幕鎖定導致中斷。

## 🛠️ 開發伺服器卡死排查與自救 (Troubleshooting & Self-Help)

在本地進行開發或使用本機網頁伺服器（預設 Port `8001`）時，偶爾會因為舊的 Python 背景行程未正常釋放，導致網頁在瀏覽器中開啟時「卡住不動」或「轉圈圈」。

### 🛡️ 預防方案 (一鍵修復)
我們提供了一個自動化腳本。當您遇到卡住或要開啟伺服器時，請直接執行：
* 👉 **`start_dev_server.bat`** (Windows 批次檔)
* 它會自動執行以下步驟：
  1. 檢查 Port `8001` 是否已被佔用。
  2. 若有舊的卡死伺服器，自動對其發送 `taskkill` 強制關閉。
  3. 重新開啟一個獨立的 CMD 視窗執行新的 Python 伺服器。

---

### 🚨 手動自救指南
如果您想手動排查並修復此問題，請根據您的系統類型使用以下指令：

#### 💻 Windows 環境 (PowerShell)
1. **檢查 Port 8001 佔用狀況**（找出 PID 行程代碼）：
   ```powershell
   Get-NetTCPConnection -LocalPort 8001 -ErrorAction SilentlyContinue | Select-Object OwningProcess
   ```
2. **強制結束卡死的行程**（假設查出來的 PID 為 `19092`）：
   ```powershell
   Stop-Process -Id 19092 -Force
   ```

#### 💻 Windows 環境 (CMD / 傳統命令提示字元)
1. **檢查 Port 8001 佔用狀況**：
   ```cmd
   netstat -aon | findstr :8001 | findstr LISTENING
   ```
   *(輸出的最後一欄即為行程 PID)*
